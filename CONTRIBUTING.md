# Contributing

This is built in public, and small contributions are the best kind. You don't need to be an expert — you need to have felt the pain.

## Ways to help

- **🧩 Suggest a rule.** Something specific that reliably makes design better. Open an issue titled `Rule: …`. Best rules are copy-able ("hero ≈ 4–6× body"), not vague ("use good hierarchy").
- **🔍 Add a teardown.** Pick a portfolio or case study that does *one* thing brilliantly. Describe *how* — with numbers/quotes where you can. **Link to it; don't upload screenshots you don't own.**
- **🥊 Argue with a rule.** Think one's wrong or too rigid? Open an issue with a counter-example. Disagreement makes this better.
- **✍️ Improve the writing.** Tighten a section, fix a typo, add a before→after example.

## Ground rules

- **Specific beats general.** A real example beats an opinion.
- **No redistributing others' images.** Reference examples by name and link. Only add images you made or own.
- **Keep the voice plain.** No filler, no corporate-speak. Say the thing.
- **Credit sources.** If a rule came from someone, name them.

## How

Fork → branch → PR, or just open an issue if you're not sure. Every idea gets a reply.
